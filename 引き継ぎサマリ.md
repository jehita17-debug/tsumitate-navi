# つみたてNAVI - 引き継ぎサマリ

## プロジェクト概要

投資初心者向けアフィリエイトサイト「つみたてNAVI」を構築し、GitHub Pagesで公開。毎朝7時(JST)に GitHub Actions が Claude API で記事を自動生成・公開する仕組みまで完成済み。

## 主要URL

- **公開サイト**: https://jehita17-debug.github.io/tsumitate-navi/
- **GitHubリポジトリ**: https://github.com/jehita17-debug/tsumitate-navi
- **Actions実行履歴**: https://github.com/jehita17-debug/tsumitate-navi/actions

## ファイル構成

アフィリエイトサイト/

## 構成の要点

- 静的HTML/CSSのみ、フレームワーク不使用
- 対象ジャンル: NISA / つみたて投資 / 米国株 / 日本株
- ターゲット: 投資初心者
- 1記事1,500〜2,500文字、図表・シミュレーション・リスク注記入り
- 広告枠は `<div class="affiliate-card">` で統一（PRバッジ自動表示）
- 投資勧誘ではない／自己責任の注記を全記事に挿入

## 自動投稿の仕組み

- **メイン**: GitHub Actions（cron: `0 22 * * *` = UTC 22:00 = JST 07:00）
  - シークレット `ANTHROPIC_API_KEY` 登録済み（Claude Sonnet 4を使用）
  - 1記事あたりAPI料金 3〜8円
- **バックアップ**: Cowork スケジュールタスク `tsumitate-navi-daily-article`（毎朝7時）

## ローカル環境

- GitHub Desktop で `C:\Users\user\Documents\GitHub\tsumitate-navi` にクローン済み（OneDrive配下は元データ）
- 編集時はDocuments配下で作業、Commit→PushしてGitHubに反映
- OneDrive側のフォルダは「常にこのデバイスに保持する」設定で同期問題を回避済み

## ハマったポイント（参考）

- OneDrive配下のフォルダはブラウザのドラッグ＆ドロップで弾かれる
   → 解決：GitHub Desktopでクローン先をDocuments配下にする
- `.github` 等のドット始まりフォルダはドラッグでスキップされやすい
   → 解決：GitHub Desktop経由なら問題なし
- 無料プランでPages使うにはリポジトリをPublic必須
